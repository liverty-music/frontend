# Poseidon constants — license provenance

`poseidon.circom` and `poseidon_constants.circom` in this directory are **MIT-licensed**
and replace the GPL-3.0 `circomlib` Poseidon that the circuit previously `include`d. This
removes copyleft code from the distributed PWA bundle (the published `ticketcheck.wasm` is
recompiled from these MIT sources; see `../../src/services/proof-service.ts` integrity
manifest).

## Why this is a clean-room reimplementation, not a relicensed copy

`circomlib`'s Poseidon constants are **not** circomlib-original data — circomlib's own
`poseidon_constants.circom` header attributes them to the public Poseidon authors' reference
generator:

> Parameters are generated by a reference script
> `https://extgit.iaik.tugraz.at/krypto/hadeshash/-/blob/master/code/generate_parameters_grain.sage`
> Used like so: `sage generate_parameters_grain.sage 1 0 254 2 8 56 0x30644e72…0001`

The constants are therefore deterministic outputs of a public academic algorithm (the Grain
LFSR + Cauchy MDS from the Poseidon paper, eprint 2019/458) on fixed public parameters
(BN254, x^5 S-box, R_F=8, R_P per width). `tools/generate.py` is an **independent MIT
implementation** of that algorithm (Grain LFSR + the Appendix-B sparse-MDS optimization). The
circom template (`poseidon.circom`) is independently authored.

## Reproduce + verify

```bash
# Regenerate the constants from the clean-room generator
npm run gen:constants            # python3 tools/emit_circom.py -> poseidon_constants.circom

# Verify byte-identity against a circomlib checkout (the GPL oracle), if available
npm run verify:constants -- path/to/circomlib/circuits/poseidon_constants.circom

# Recompile the circuit and confirm the R1CS is byte-identical (so the existing .zkey
# and backend verification_key.json are reused unchanged)
npm run build:wasm
sha256sum build/ticketcheck.r1cs
# => a8a5a293b869522b47b78fc4043007007945a5d26732379ae1e5fe6c2ba846f2
```

## Verified equivalence (2026-06-05)

- Constants byte-identical to circomlib for the widths the circuit uses: **t=2** (`Poseidon(1)`)
  and **t=3** (`Poseidon(2)`) — C/M/P/S all match.
- Recompiled R1CS sha256 = `a8a5a293…846f2`, identical to the R1CS behind the committed
  `ticketcheck.zkey` (`nVars=5336, nPublic=3, domainSize=8192`) → `.zkey` and backend vkey
  reused unchanged.
- Hash outputs match `circomlibjs` `buildPoseidon`, so existing identity commitments stay valid.
- A proof generated with the recompiled `ticketcheck.wasm` + reused `.zkey` verifies through
  the backend's `circom2gnark` → `gnark.VerifyProof` path against the existing vkey.
